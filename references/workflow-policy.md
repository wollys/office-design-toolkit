# Office Document Workflow Policy (Mandatory)

## Core Sequence
Always execute in this order for DOCX/XLSX/PPTX/PDF tasks:
1. Content Draft
2. Structure Design
3. Visual Design & Polish
4. QA & Delivery

Do not skip phases unless user explicitly asks to skip.

## Phase 1 — Content Draft
- Clarify objective, audience, expected output.
- Produce complete content first (outline + key details + CTA/KPI where relevant).
- Validate factual consistency and completeness.
- Keep style instructions internal; do not write them into output body.

## Phase 2 — Structure Design
- Convert draft into information architecture:
  - Title
  - Executive summary / context
  - Main sections
  - Action plan / KPI / timeline
  - Notes / appendix
- Use tables only when they improve scanning.
- Prioritize fast reading in 30–60 seconds.

## Phase 3 — Visual Design & Polish
- Apply approved style pack and typography hierarchy.
- Enforce consistency of spacing, alignment, and color usage.
- Keep readability above decoration.
- Never inject internal style guideline text into user-facing content.

## Phase 4 — QA & Delivery
- Content QA: complete, coherent, actionable.
- Structure QA: hierarchy clear, no orphan sections.
- Design QA: consistent style tokens, table hierarchy, contrast.
- Technical QA: file opens cleanly in Office viewers.
